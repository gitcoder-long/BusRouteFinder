# 🚌 公交路线查询系统

> 基于 C 语言的控制台公交路线规划工具，支持随机生成 / 文件导入路线数据，
> 利用 **Dijkstra 算法** 计算两站点间**总耗时最短且换乘次数最少**的最优乘车方案。
>
> 本项目是在过去写的代码基础上进行的**重构与优化**，改进了代码结构、修复了潜在问题，并提升了可维护性。

## ✨ 功能特性

- **双数据源**：随机生成 5~10 条路线，或从 `./routes` 文件加载自定义线路
- **智能路径规划**：优先直达，无法直达时自动给出换乘次数最少的方案
- **人性化输出**：展示每段乘坐的路线名、方向（正向/反向）和段内耗时
- **数据持久化**：支持将生成/编辑的路线保存至文件，下次直接加载

## 🛠️ 运行环境

- Windows 操作系统
- 支持 C99 的编译器（GCC / MSVC 均可）
- 无需第三方库，标准 C 和 `windows.h` 已包含

## 🚀 快速开始

### 编译
```bash
gcc main.c -o bus_route.exe
```
> 若使用 MSVC：`cl main.c`，生成 `main.exe`。

### 运行
```bash
bus_route.exe
```
推荐在 **命令提示符** 或 **PowerShell** 中运行，以获得最佳清屏效果。

## 📂 自定义路线文件格式 (`./routes`)

若想自己造数据，按以下格式放入项目目录即可：
```
<路线总数> <站点总数>
<路线名> <站点数量> : <站点1> <时间1> <站点2> <时间2> ...
```
示例：
```
9 15
Route1 5 : S3 5 S4 10 S6 25 S11 5 S12
Route2 8 : S1 10 S3 15 S6 5 S7 5 S8 5 S9 25 S14 5 S15
...
```
**注意：**
- 路线名 ≤ 10 字符，站点名格式 `S1`~`S15`，≤4 字符
- 相邻时间建议为正整数（分钟）
- 路线总数 5~10，站点总数 10~15

## 📖 使用说明

1. 启动后选择数据来源：  
   `1` 从文件加载 | `2` 随机生成 | `3` 退出
2. 进入查询界面，输入起点和终点（如 `S1` `S2`，或输入 `exit` 退出）
3. 查看详细乘车方案
4. 可继续查询，或在退出时保存数据到文件

## ⚠️ 注意事项

- 随机生成时，相邻站点默认耗时 5 分钟；文件可自定义不同耗时
- 输入站点必须严格遵循 `S1`~`S15` 格式，且不能超出当前数据范围
- 部分 IDE 终端出现输入字符回显两次，请将 `main` 函数中的 `SetConsoleCP` 和 `SetConsoleOutputCP` 注释掉即可

## 🧠 技术亮点

- 将公交路网建模为 **带权无向图**，邻接矩阵存储
- **Dijkstra 算法** 求最短时间，同时记录换乘次数进行**双目标优化**
- 自主设计 `calculate_transfer` 函数，在松弛时动态判断换乘增加
- 分段输出时通过贪心匹配**最长连续路段**，给出符合直觉的乘车指引
