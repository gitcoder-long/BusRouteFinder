# 公交路线查询系统

一个基于 **状态图 Dijkstra 算法** 的公交路线规划工具，使用 C 语言编写。  
能根据随机生成或文件加载的公交网络，计算两站点间 **时间最短、换乘次数最少** 的出行方案，并输出详细的乘车指引。

## ✨ 功能特色
- **灵活的数据来源**：支持从文件加载路线，也可随机生成符合约束的公交网络。
- **精确的最优路径**：在「站点×路线」的状态空间上运行 Dijkstra，天然区分同站换乘与沿途行驶，结果严格保证时间最短、换乘最少。
- **清晰的乘车指引**：自动识别直达路线；需换乘时给出每段乘坐的路线名、方向、耗时。
- **友好的交互**：循环查询、随时退出、可选择保存数据，输入具有完善的错误处理。

## 📖 算法原理
将“当前站点 + 当前所乘路线”定义为一个状态。状态之间的转移有两种：
1. **沿线移动**：在当前路线上正向或反向移动到相邻站点，时间增加边权，换乘次数不变。
2. **换乘**：在站点更换到其他存在该站的路线，时间不增加，换乘次数加 1（首次乘车不计换乘）。

在由所有状态构成的有向图上运行 Dijkstra 算法，起点状态为“起始站点 + 未乘车”，终点为“目标站点 + 任意路线”，最终倒推出经过的状态序列，再合成为乘车方案。

## 🧱 编译与运行
### 环境要求
- Windows 操作系统（使用了 `windows.h` 的控制台功能）
- 支持 C99 或更新标准的编译器（如 GCC、MSVC）
- 目前仅支持Windows（未测试其他平台）
### 编译
```bash
gcc main.c -o bus_route.exe
```

### 运行
直接双击或在命令行中执行：
```bash
bus_route.exe
```
> 推荐在 **命令提示符（CMD）或 Windows Terminal** 中运行，以获得最佳清屏效果。

## 🗺️ 使用说明
1. 启动程序，选择数据来源：  
   - `1`：从文件 `routes`（程序所在目录）加载路线  
   - `2`：随机生成路线数据  
   - `3`：退出  
2. 进入路径查询界面后，输入起点和终点（如 `S1`、`S12`），按回车。  
   - 输入 `exit` 可随时退出站点输入。  
3. 查看输出的乘车方案，包括路径、总用时、分段详情。  
4. 可选择是否继续查询；退出时若数据尚未保存，程序会询问是否保存到文件。

## 📄 路线文件格式 (`routes`)
若使用自定义数据，请按以下格式准备纯文本文件，存放在程序相同目录下：

```
<路线总数> <站点总数>
<路线名> <站点数> : <站点1> <耗时1> <站点2> <耗时2> ... <站点N>
...
```

示例：
```
9 15
Route1 5 : S3 5 S4 10 S6 25 S11 5 S12
Route2 8 : S1 10 S3 15 S6 5 S7 5 S8 5 S9 25 S14 5 S15
```

**约束：**
- 路线总数：5 ~ 10  
- 站点总数：10 ~ 15  
- 每条路线站点数：5 ~ 10  
- 站点名格式：`S1` ~ `S15`，长度不超过 4 字符  
- 相邻耗时建议为正整数（单位：分钟）
